# План деплоя Ish-Top на VPS сервер (Contabo)

Этот документ описывает пошаговый план переноса проекта Ish-Top (Frontend: React/Vite, Backend: FastAPI, БД: PostgreSQL) на выделенный VPS сервер от Contabo (рекомендуемая ОС: Ubuntu 22.04 / 24.04 LTS).

## User Review Required

> [!IMPORTANT]  
> **Выбор архитектуры развертывания**
> Ниже представлен план развертывания с использованием **нативных сервисов Linux** (Nginx, Systemd, локальный PostgreSQL). Это дает максимальный контроль и производительность. 
> *Альтернативный вариант:* использование **Docker / Docker Compose**. Это проще в поддержке, но требует немного больше оперативной памяти. Напишите, если вы предпочитаете вариант с Docker.

## Этапы развертывания

### Этап 1: Базовая настройка сервера (VPS)
1. Подключение к серверу по SSH (`ssh root@ip_вашего_сервера`).
2. Обновление системы: `apt update && apt upgrade -y`.
3. Установка базовых утилит: `curl`, `git`, `ufw` (настройка файрвола), `nginx`.
4. Настройка файрвола: открытие портов 22 (SSH), 80 (HTTP), 443 (HTTPS).
5. Создание отдельного пользователя для работы приложения (из соображений безопасности).

### Этап 2: Настройка базы данных (PostgreSQL)
1. Установка PostgreSQL: `apt install postgresql postgresql-contrib`.
2. Вход в консоль psql и создание пользователя и базы данных:
   - Создание пользователя: `CREATE USER ish_top_user WITH PASSWORD 'strong_password';`
   - Создание БД: `CREATE DATABASE ish_top_db OWNER ish_top_user;`
3. Настройка кодировки и прав доступа.

### Этап 3: Деплой Backend (FastAPI)
1. Установка Python и Pip: `apt install python3-pip python3-venv`.
2. Клонирование репозитория проекта в папку на сервере (например, `/var/www/ish-top`).
3. Создание и активация виртуального окружения (`python3 -m venv venv`).
4. Установка зависимостей из `backend/requirements.txt`.
5. Создание файла `backend/.env` с боевыми настройками (URL к локальной БД, секретные ключи, токен Telegram бота).
6. Выполнение миграций Alembic: `alembic upgrade head`.
7. **Настройка Systemd:** Создание сервиса `/etc/systemd/system/ishtop-backend.service`, который будет запускать бэкенд через `Gunicorn` (с uvicorn workers) в фоновом режиме, автоматически перезапуская его при падениях или перезагрузке сервера.

### Этап 4: Деплой Frontend (React + Vite)
1. Установка Node.js (рекомендуется v20+) и npm.
2. Переход в папку `frontend` и установка зависимостей: `npm install`.
3. Сборка проекта для продакшена: `npm run build`. 
   *(После этого появится папка `dist` с готовой статикой).*
4. Копирование папки `dist` в директорию, откуда ее будет раздавать Nginx (например, `/var/www/ish-top/frontend/dist`).

### Этап 5: Настройка Nginx (Веб-сервер и Реверс-прокси)
1. Создание конфигурационного файла Nginx (`/etc/nginx/sites-available/ishtop`).
2. Настройка Nginx:
   - Раздача статических файлов фронтенда (`root /var/www/ish-top/frontend/dist;`).
   - Настройка маршрутизации для React Frontend (fallback на `index.html` для SPA).
   - Раздача статических файлов бэкенда (например, медиа загрузок) из папки `backend/static`.
   - Проксирование API запросов (`/api/`) на адрес, где работает FastAPI (например, `http://127.0.0.1:8000`).
3. Активация конфигурации и перезапуск Nginx.

### Этап 6: Установка SSL сертификатов (HTTPS)
1. Привязка доменного имени к IP адресу VPS (на стороне регистратора домена).
2. Установка Certbot: `apt install certbot python3-certbot-nginx`.
3. Генерация бесплатных SSL сертификатов от Let's Encrypt: `certbot --nginx -d it-top.uz -d www.it-top.uz`.
4. Проверка автоматического обновления сертификата.
